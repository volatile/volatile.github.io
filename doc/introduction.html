Introduction

Volatile is a web framework for Go (golang).

Volatile Core is the perfect foundation for any web app as it's designed to have the best balance between readability, flexibility and performance.

It provides a pure handlers (or middlewares) stack so you can perform actions downstream, then filter and manipulate the response upstream.

No handlers or helpers are bundled in the Core: it does one thing and does it well.
You can find official packages below.

The handlers stack

The handlers stack is the backbone of Volatile. Everything revolves around it.

The handler

A handler is simply a function that receives a context.
It has the possibility to break the sream or to continue with the next handler in the stack.

To add a handler to the stack, just use the Core's Use function:

core.Use(func(c *core.Context) {
    c.Next() // Go to the next handler in the stack.
})

core.Use(func(c *core.Context) {
    // No c.Next(). The handler stream ends here.
})

The context

A context is created for each request and throwed inside the handlers stack.
It encapsulates the classics: the *http.Request and the http.ResponseWriter from the standard net/http package.
